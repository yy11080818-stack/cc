# 阡陌通AI应用可视化大屏需求分析

> 分析时间：2026年4月27日
> 需求内容：AI方案编制功能 + AI隐患识别功能
> 需求类型：功能定位 + 平台归属 + 可视化大屏设计

---

## 📋 需求理解

### 核心需求
1. **新增两个AI功能**：AI方案编制 + AI隐患识别
2. **平台归属**：选择合适的模块位置安放，不是融合所有模块
3. **可视化大屏**：设计数据展示大屏，重点是**数据呈现**，不是业务操作

### 设计原则
- ✅ 只展示数据结果，不展示业务流程
- ✅ 突出关键指标（KPI），支持决策
- ✅ 支持多层级查看（集团级 → 项目级 → 标段级）
- ✅ 关键数据实时更新

## 第一部分：AI方案编制功能定位与数据定义

### 1.1 功能定位

**AI方案编制是什么**：
- 通过AI技术（大语言模型+知识库）辅助施工方案编制
- 核心能力：方案生成、方案审核、知识检索、模板管理
- 用户：项目总工、技术部长、技术员
- **关键**：这是一个**工具型功能**，帮助技术人员提高效率

### 1.2 核心数据指标

#### 方案编制指标
| 指标名称 | 说明 | 数据类型 |
|---------|------|---------|
| **方案编制总数** | 累计编制方案数量 | 数值 |
| **本月编制数** | 当月编制方案数量 | 数值 |
| **方案类型分布** | 按桥梁/隧道/路基/路面等分类 | 饼图 |
| **编制时长对比** | AI辅助前后时长对比 | 柱状图 |
| **效率提升率** | (传统时长-AI时长)/传统时长 | 百分比 |

#### 方案审核指标
| 指标名称 | 说明 | 数据类型 |
|---------|------|---------|
| **审核总数** | 累计审核方案数量 | 数值 |
| **审核通过率** | 通过数量/审核总数 | 百分比 |
| **问题类型分布** | 合规性问题/完整性问题/工艺先进性问题 | 饼图 |
| **AI审核准确率** | AI识别问题准确率 | 百分比 |

#### 知识库指标
| 指标名称 | 说明 | 数据类型 |
|---------|------|---------|
| **历史方案数量** | 知识库中历史方案总数 | 数值 |
| **方案模板数量** | 标准化方案模板数量 | 数值 |
| **规范标准数量** | 行业规范标准数量 | 数值 |
| **知识库使用频次** | 知识检索次数 | 数值 |
| **高频模板TOP10** | 使用最多的10个模板 | 列表 |

### 1.3 可视化需求

**AI方案编制可视化大屏需要展示**：
1. **整体概况**：编制总数、审核总数、效率提升率、知识库规模
2. **编制趋势**：按月/季度统计编制数量趋势图
3. **审核分析**：审核通过率、问题类型分布、AI审核准确率
4. **知识库分析**：模板使用频次、知识检索热词
5. **效率对比**：AI辅助前后效率对比（编制时长、审核时长）

## 第二部分：AI隐患识别功能定位与数据定义

### 2.1 功能定位

**AI隐患识别是什么**：
- 通过AI技术（计算机视觉+知识库）自动识别施工现场安全隐患
- 核心能力：视频流识别、随手拍识别、隐患预警、隐患台账
- 用户：安全员、安全经理、集团安全总监
- **关键**：这是一个**安全监测功能**，帮助安全管理

### 2.2 核心数据指标

#### 隐患识别指标
| 指标名称 | 说明 | 数据类型 |
|---------|------|---------|
| **今日识别隐患数** | 当日AI识别的隐患数量 | 数值 |
| **累计识别隐患数** | 累计AI识别的隐患总数 | 数值 |
| **识别准确率** | AI识别准确率（≥85%） | 百分比 |
| **隐患类型分布** | 12大类隐患类型分布 | 饼图 |
| **隐患等级分布** | 高级/中级/初级隐患分布 | 饼图 |
| **识别趋势** | 按日/周/月统计识别数量 | 折线图 |

#### 隐患处置指标
| 指标名称 | 说明 | 数据类型 |
|---------|------|---------|
| **今日整改数** | 当日完成整改的隐患数量 | 数值 |
| **整改完成率** | 已整改数/识别总数 | 百分比 |
| **平均整改时长** | 从识别到整改的平均时长 | 小时/天 |
| **逾期未整改数** | 超期未整改的隐患数量 | 数值 |
| **整改率趋势** | 按周/月统计整改率 | 折线图 |

#### 监测覆盖指标
| 指标名称 | 说明 | 数据类型 |
|---------|------|---------|
| **在线摄像头数** | 在线监控摄像头数量 | 数值 |
| **巡检覆盖率** | AI巡检覆盖的区域比例 | 百分比 |
| **今日告警数** | 今日AI识别告警数量 | 数值 |
| **告警响应率** | 已响应告警数/告警总数 | 百分比 |

#### 风险分析指标
| 指标名称 | 说明 | 数据类型 |
|---------|------|---------|
| **高风险区域TOP10** | 隐患频发的Top10区域 | 列表+地图标注 |
| **高频隐患类型TOP5** | 出现最多的5类隐患 | 列表 |
| **隐患趋势分析** | 隐患数量变化趋势 | 折线图 |
| **时段分布分析** | 隐患高发时段分析 | 热力图 |

### 2.3 可视化需求

**AI隐患识别可视化大屏需要展示**：
1. **实时态势**：今日识别数、待整改数、在线摄像头、实时告警滚动
2. **隐患分布**：隐患类型分布、等级分布、区域分布（地图）
3. **处置分析**：整改完成率、平均整改时长、逾期未整改
4. **趋势分析**：隐患趋势、整改率趋势、时段分布
5. **风险预警**：高风险区域、高频隐患类型、异常告警

## 第三部分：平台模块归属

### 3.1 阡陌通平台模块回顾

阡陌通平台现有5大中心：
1. **AI大脑**：智能交互、数据查询、知识问答
2. **数字孪生中心**：BIM+GIS可视化、进度回溯
3. **数字建管中心**：设计、征拆、技术、质量、进度、计量、星级评价
4. **数字建造中心**：路基、路面、桥梁、隧道、试验室、拌合站
5. **数字安全中心**：工点监测、高边坡、塔吊、无人机、智慧保通

### 3.2 平台归属方案

| AI功能 | 平台归属 | 位置 | 理由 |
|-------|---------|------|------|
| **AI方案编制** | **数字建管中心** | 技术管理模块（新增） | 技术管理的核心内容，用户群体一致（技术员、技术部长、项目总工） |
| **AI隐患识别** | **数字安全中心** | 工点监测模块（增强） | 智能抓拍的增强扩展，用户群体一致（安全员、安全经理、集团安全总监） |

### 3.3 归属理由

**AI方案编制 → 数字建管中心-技术管理**：
- 技术管理模块已有"项目与节段管理"、"交组模拟"、"3D可视化"
- AI方案编制是技术管理的核心内容之一
- 用户群体一致（技术员、技术部长、项目总工）

**AI隐患识别 → 数字安全中心-工点监测**：
- 工点监测模块已有"监控设备"、"实时视频"、"智能抓拍"、"预警推送"
- AI隐患识别是智能抓拍的增强和扩展
- 用户群体一致（安全员、安全经理、集团安全总监）

## 第四部分：可视化大屏框架设计

### 4.1 大屏布局框架

```
┌─────────────────────────────────────────────────────────────┐
│                     阡陌通AI应用可视化大屏                      │
├─────────────────────────────────────────────────────────────┤
│  [一级导航] AI方案编制 | AI隐患识别 | 综合视图                   │
├─────────────────────────────────────────────────────────────┤
│  [层级选择] 集团 ▼ | 项目 ▼ | 标段 ▼ | 时间范围 ▼              │
├───────────────────┬───────────────────────────────────────────┤
│  [左侧指标区]     │         [中间主要展示区]                  │
│  - 核心KPI卡片    │         - 地图/BIM可视化                 │
│  - 趋势图表       │         - 实时数据滚动                    │
│  - 分布图表       │         - 异常预警提示                    │
│  - 排行榜单       │                                           │
├───────────────────┴───────────────────────────────────────────┤
│  [底部详情区] 详情数据表格/图表（可折叠）                      │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 数据层级

**数据层级**：
1. **集团级**：展示所有项目汇总数据，支持项目筛选
2. **项目级**：展示单个项目数据，支持标段筛选
3. **标段级**：展示单个标段数据，支持工点筛选

### 4.3 实时性策略

- **实时数据**：今日识别数、实时告警（每5秒刷新）
- **准实时数据**：整改完成率、在线摄像头（每1分钟刷新）
- **静态数据**：历史趋势、类型分布（每小时刷新）

### 4.4 异常预警

- **红色闪烁**：高危隐患、逾期未整改
- **黄色提醒**：整改超时、识别准确率下降
- **绿色提示**：正常完成

## 第七部分：总结与建议

### 7.1 平台归属总结

| AI功能 | 平台归属 | 位置 | 用户群体 |
|-------|---------|------|---------|
| **AI方案编制** | **数字建管中心** | 技术管理模块（新增） | 技术员、技术部长、项目总工 |
| **AI隐患识别** | **数字安全中心** | 工点监测模块（增强） | 安全员、安全经理、集团安全总监 |

### 7.2 可视化大屏特点

**AI方案编制大屏**：
- 侧重**效率分析**（编制效率、审核效率）
- 侧重**知识管理**（模板使用、知识库规模）
- 数据更新频率：每小时（非实时性数据）

**AI隐患识别大屏**：
- 侧重**实时监测**（今日识别、实时告警）
- 侧重**隐患分布**（GIS地图、类型分布、等级分布）
- 数据更新频率：每5秒（高实时性数据）

### 7.3 实施建议

1. **先做AI方案编制可视化大屏**
   - 数据相对静态，技术难度低
   - 价值明确（效率提升可量化）
   - 用户接受度高（辅助工具）

2. **后做AI隐患识别可视化大屏**
   - 需要实时数据流，技术难度高
   - 需要先验证AI识别准确率
   - 用户接受度取决于识别效果

3. **分阶段实施**
   - 第一阶段：数据指标定义 + 数据接口开发
   - 第二阶段：大屏界面设计 + 可视化组件开发
   - 第三阶段：数据联调 + 测试优化
